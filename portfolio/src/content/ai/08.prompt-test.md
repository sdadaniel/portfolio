## 프롬프트 테스트

### 목적
- 프롬프트(지시문) 변경에 따른 출력 품질/일관성/안전성/비용 영향을 빠르게 비교합니다.

### 테스트 템플릿
- **테스트 이름**:
- **대상 작업**: (예: 요약, 분류, 생성, 추출, 코드리뷰 등)
- **입력(고정)**:
- **프롬프트 v1**:
- **프롬프트 v2**:
- **평가 기준**:
  - 정확성 / 누락 / 환각
  - 형식 준수(스키마/톤/길이)
  - 재현성(여러 번 실행 시 일관성)
  - 안전/정책 위반 가능성
  - 비용/지연
- **결과**:
  - v1:
  - v2:
- **결론 / 다음 액션**:

### 체크리스트
- 입력은 가능한 한 고정(동일)하고, 변수(온도/샘플링/툴 사용 여부)를 명시합니다.
- “좋다/나쁘다” 대신 **구체적인 실패 케이스**(예: 어떤 문장이 틀렸는지)를 기록합니다.
- 필요하면 대표 샘플 3~10개를 선정해 회귀 테스트로 재사용합니다.
