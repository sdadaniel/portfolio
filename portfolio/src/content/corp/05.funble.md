# 펀블

**2025.01 - 2025.10** · 정규직 · 금융권 하이브리드 앱(WebView) 운영 및 개발 주도

## Summary

금융 서비스 환경에서 **하이브리드 앱(App·WebView)** 구조를 안정화하고, **인증·딥링크·실시간 기능**과 **운영형 UI(에디터·설정)**를 프로젝트 단위로 개선해 사용자 경험과 운영 효율을 함께 끌어올린 경험.

## 1. 하이브리드 앱 인증·딥링크·WebView–Native 연동 안정화

### Problem

- 금융 규정·보안을 고려한 **인증/본인인증**과 **WebView–Native 연동**이 복잡하고, iOS/Android 및 신규 OS 대응이 지속적으로 필요함
- 앱↔웹 이동 및 측정을 위한 **딥링크 구조**가 정리되지 않으면 운영·분석·CS 대응이 흔들림

### Approach

- **React Native** 앱 기능 개발 및 네이티브 모듈 연동
- iOS **네이티브 키패드** 모듈 커스터마이징 및 신규 OS 대응
- **코스콤 클라우드 인증서** 로그인·**법인 인증** 플로우 구현
- **Airbridge** 기반 딥링크 구조 구축(진입 경로/파라미터 규칙 정리)
- **WebView–Native 본인인증** 연동을 통합·안정화(플로우 단일화)

### Result

- 인증·이동 경로를 **예측 가능한 흐름**으로 정리해 하이브리드 서비스 **운영 안정화** 및 **구조 개선**에 기여
- 신규 OS/디바이스 이슈 대응 부담을 낮추고, 연동 변경 시 영향 범위를 더 명확히 파악할 수 있게 됨

## 2. WebView(Vue3) 번들·로딩 최적화 및 유지보수성 개선

### Problem

- WebView(**Vue3**) 기능이 늘면서 **번들·로딩** 부담과 운영 수정 범위가 커짐

### Approach

- WebView 기반 화면 개발 및 설정·인증 로직 유지보수
- **webpack-bundle-analyzer**로 병목을 분석하고, **CJS → ESM** 전환 등으로 **번들 사이즈 최적화**

### Result

- WebView 측 **성능·유지보수** 여지를 확보하고 배포 단위별 변경 리스크를 낮추는 방향으로 개선

## 3. WebSocket 기반 실시간 채팅·거래 기능 개발

### Problem

- 실시간 커뮤니케이션/거래 흐름은 UX 요구치가 높고, 장애·지연이 발생하면 사용자 신뢰에 직접 영향을 줌

### Approach

- **WebSocket** 기반의 실시간 채팅·거래 기능을 개발하고, 운영 중 발생하는 이슈를 반영해 고도화

### Result

- 실시간 UX 요구를 충족하는 기능 라인을 안정적으로 유지·발전시킴

## 4. 디자인 시스템 구조화 및 Storybook 도입

### Problem

- 공통 UI가 흩어져 있어 화면별 구현 품질이 달라지고, 컴포넌트 재사용/검증 비용이 커짐

### Approach

- 디자인 시스템을 구조화하고 **Storybook**을 도입해 UI 컴포넌트를 문서화
- 공통 UI 패키지의 **번들 최적화**로 배포/적용 비용을 낮춤

### Result

- 디자인·개발·운영이 같은 컴포넌트 기준을 보도록 정리되어 UI 일관성과 재사용성이 개선됨

## 5. 운영용 에디터(Tiptap)·설정 및 도메인 화면 개선

### Problem

- 운영 콘텐츠 작성과 설정 변경 흐름에서 UX 제약이 있었고, 도메인 특화 화면(예: **수익자총회**)에서는 판매·결과 반영 로직 정리가 필요함

### Approach

- **Tiptap** 에디터를 커스텀(테이블 균등 너비, 이미지 100% 폭, 캡션, 플레이스홀더, 문단 전환 등)하고 링크 버튼·텍스트 배경색 등 운영 UX를 보강
- 설정 핵심 화면을 **전면 리뉴얼**(IA·UX 재정비)
- 수익자총회 **페이지 구조 개선** 및 판매·결과 반영 로직 구현

### Result

- 에디터·설정·도메인 화면의 반복 이슈를 줄이고, 운영 효율을 높이는 방향으로 개선함

## Tech Stack

React, React Native, WebView(Vue3), TypeScript, Storybook, WebSocket, Airbridge 등(이력서 기재 스택 기준)
