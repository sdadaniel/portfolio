# 펀블

**2025.01 - 2025.10** · 정규직 · 금융권 하이브리드 앱(WebView) 운영 및 개발 주도

## Summary

금융 서비스 환경에서 **앱·WebView**를 아우르는 하이브리드 구조를 안정화하고, **네이티브·인증·딥링크·실시간 거래·운영형 UI(에디터·설정)**까지 프론트 영역을 넓게 맡아 구조 개선과 사용자·운영 효율을 동시에 맞춘 경험.

## 1. 하이브리드 앱·금융 인증·연동

### Problem

- 금권 규정·보안을 고려한 **인증·WebView–Native 연동**이 복잡하고, iOS/Android·신규 OS 대응이 필요함
- 앱·웹 간 이동·측정을 위한 **딥링크·플로우** 정리 시점이 요구됨

### Approach

- **React Native** 앱 기능 개발 및 네이티브 모듈 연동
- iOS **네이티브 키패드** 모듈 커스터마이징 및 신규 OS 대응
- **코스콤 클라우드 인증서** 로그인·**법인 인증** 플로우 구현
- **Airbridge** 기반 딥링크 구조 구축
- **WebView–Native 본인인증** 통합 및 안정화

### Result

- 하이브리드 서비스 **운영 안정화** 및 **구조 개선**에 기여하여 인증·이동 경로를 예측 가능한 형태로 정리

## 2. WebView 프론트엔드·성능

### Problem

- WebView(**Vue3**) 기능이 늘면서 **번들·로딩** 부담과 운영 수정 범위가 커짐

### Approach

- WebView 기반 화면 개발 및 설정·인증 로직 수정
- **webpack-bundle-analyzer**로 분석하고 **CJS → ESM** 전환 등 **번들 사이즈 최적화**

### Result

- WebView 측 **성능·유지보수** 여지를 확보하고 배포 단위별 변경 리스크를 낮추는 방향으로 개선

## 3. 실시간 채팅·거래

### 개요

금융 서비스 맥락에서 **WebSocket** 기반 **실시간 채팅·거래** 기능을 개발하고 고도화.

### 결과

- 실시간 UX 요구를 충족하는 기능 라인을 유지·발전시킴

## 4. 디자인 시스템·에디터·설정·도메인 화면

### Problem

- 공통 UI·**운영용 에디터**·**설정** 화면이 흩어져 있어 일관성과 운영 편의가 떨어짐
- **수익자총회** 등 도메인 특화 화면에서 판매·결과 반영 로직 정리가 필요함

### Approach

- 디자인 시스템 구조화 및 **Storybook** 도입, UI 컴포넌트 문서화·공통 패키지 **번들 최적화**
- **Tiptap** 에디터 커스텀(테이블 균등 너비, 이미지 100% 폭, 캡션, 플레이스홀더, 문단 전환 등), 링크 버튼·텍스트 배경색 등 운영 UX 보강
- 설정 핵심 화면 **전면 리뉴얼**(IA·UX 재정비)
- 수익자총회 **페이지 구조 개선** 및 판매·결과 반영 로직 구현

### Result

- 디자인·개발·운영이 같은 컴포넌트 기준을 보도록 정리되고, 에디터·설정·도메인 화면의 **반복 이슈 감소**에 기여

## Tech Stack

React, React Native, WebView(Vue3), TypeScript, Storybook, WebSocket, Airbridge 등(이력서 기재 스택 기준)
